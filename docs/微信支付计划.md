## 微信小程序支付接入计划

> 目标：在当前校园跑腿小程序中接入**微信小程序支付**，所有订单（代取、跑腿等）通过统一的微信支付入口完成支付。

---

### 一、你需要先完成的准备工作

1. **微信支付商户配置**
   - 登录微信支付商户平台，确认已开通「小程序支付」能力。
   - 准备好以下参数（后面会写进项目配置）：
     - `mchid`：微信支付商户号
     - `appid`：当前小程序 AppID（与 `manifest.json` 中 `mp-weixin.appid` 一致）
     - `APIv3 Key`：商户平台设置的 v3 密钥
     - 商户私钥：`apiclient_key.pem`
     - 平台证书序列号 `serial_no` 及证书内容
2. **配置支付回调地址（这一项才需要域名，普通小程序支付本身不需要）**
   - 小程序端通过 `wx.requestPayment` 拉起收银台本身**不需要域名**，只依赖 `appid/mchid/prepay_id` 等参数即可（见[官方指引](https://pay.weixin.qq.com/doc/v3/merchant/4012791911)中“商户调起支付”部分）。
   - 只有“支付结果通知”这一步，才需要在商户平台配置 云平台给你的 HTTPS 访问地址 回调 URL，例如：`https://你的域名/pay/wechat/notify`。
   - 后续会把这个回调 URL 指向 uniCloud 的 HTTP 云函数，用于接收微信支付的结果通知。

### 三、云端：创建微信支付云对象 `payment-service`

> 目录：`uniCloud-aliyun/cloudfunctions/payment-service/`

1. **基础结构**
   - 创建云对象文件 `index.obj.js` 和 `package.json`。
   - 在 `_before` 中：
     - 使用 `uni-id-common` 校验登录，获取 `uid`。
     - 读取 `wechat-pay.json` 配置。
2. **方法一：`createJsapiOrder({ orderId })`**
   - 功能：根据业务订单创建微信 JSAPI 订单，返回前端 `wx.requestPayment` 所需参数。
   - 步骤：
     1. 从 `orders` 表查 `orderId`，确认订单存在且属于当前 `uid`，状态为未支付。
     2. 校验金额（以订单记录为准）。
     3. 从 `uni-id-users` 中获取当前用户在本小程序下的 `openid`（`wx_openid.mp`）。
     4. 生成商户订单号 `out_trade_no`，写入订单与 `transactions` 表（新增一条 type=`pay`、status=`pending` 的流水）。
     5. 构造 `/v3/pay/transactions/jsapi` 请求体：
        - `appid/mchid/out_trade_no/description/notify_url/amount.total/payer.openid`
     6. 使用商户私钥对请求头签名，调用微信支付接口。
     7. 解析返回结果中的 `prepay_id`。
     8. 生成小程序支付参数：
        - `timeStamp/nonceStr/package('prepay_id=...')/signType/paySign`
     9. 返回给前端。
3. **方法二：`queryOrder({ outTradeNo })`**
   - 封装 `/v3/pay/transactions/out-trade-no/{out_trade_no}`。
   - 提供给前端轮询或后台补偿逻辑使用。
4. **方法三：`closeOrder({ outTradeNo })`（可选）**
   - 封装关闭订单接口，用于超时未支付订单的关单处理。
5. **方法四：`refund({ orderId, amount, reason })`（后续扩展）**
   - 封装退款接口，对接现有 `transactions` 表。

### 四、云端：创建支付结果回调 HTTP 云函数

> 目录示例：`uniCloud-aliyun/cloudfunctions/wechat-pay-notify`（**普通云函数**，非云对象）

#### 4.1 创建云函数并配置 URL 化

1. **在 HBuilderX 中创建普通云函数**
   - 右键 `uniCloud-aliyun/cloudfunctions` → 新建 → 云函数 → 输入名称 `wechat-pay-notify`
   - 创建 `index.js` 作为入口文件（普通云函数使用 `exports.main`，不是云对象的 `module.exports`）

2. **在 uniCloud Web 控制台配置 URL 化**
   - 登录 [uniCloud 后台](https://unicloud.dcloud.net.cn/)，选择服务空间
   - 进入【云函数】→ 找到 `wechat-pay-notify` → 点击【详情】
   - 在【云函数 URL 化】中设置访问路径，例如：`/wechat-pay-notify`
   - 如果提示“URL 化不可用”，需要先绑定域名：
     - 点击【域名绑定】→ 使用**平台赠送的默认域名**（如 `xxx.bspapp.com`）或绑定自定义域名
     - 绑定成功后，返回云函数详情页配置 URL 化路径

3. **获取 HTTPS 访问地址**
   - 配置完成后，访问地址格式为：`https://你的域名/http/wechat-pay-notify`
   - 例如：`https://xxx.bspapp.com/http/wechat-pay-notify`
   - **将此地址填入**：
     - `wechat-pay.json` 的 `notify_url` 字段
     - 微信商户平台的【支付配置】→【支付回调 URL】

#### 4.2 云函数实现要点

1. **接收微信回调**
   - 普通云函数通过 `event` 参数获取请求信息
   - 注意：请求体可能是 base64 编码的，需要判断 `event.isBase64Encoded`

2. **验签**
   - 从 `event.headers` 读取：
     - `Wechatpay-Signature`
     - `Wechatpay-Timestamp`
     - `Wechatpay-Nonce`
     - `Wechatpay-Serial`
   - 根据 `serial` 查找平台证书，使用证书公钥验证签名

3. **解密报文**
   - 先处理 base64 编码（如需要）：
     ```js
     let body = event.body
     if(event.isBase64Encoded) {
       body = Buffer.from(body, 'base64').toString('utf8')
     }
     ```
   - 解析 JSON 后，使用 `APIv3 Key` 对 `resource` 字段进行 AES-GCM 解密
   - 获取交易详情：`out_trade_no/transaction_id/trade_state/amount` 等

4. **业务处理**
   - 根据 `out_trade_no` 查询本地 `transactions` / `orders`：
     - 校验金额是否一致
     - 若尚未处理：
       - 更新订单 `pay_status = 'paid'`，业务状态从 `pending_pay` → `pending_accept`
       - 更新 `transactions` 记录为 `success`，写入 `transaction_id` 等信息
   - 确保幂等（重复通知不会重复记账）

5. **返回应答（使用集成响应格式）**
   - 成功时返回：
     ```js
     return {
       statusCode: 200,
       headers: { 'content-type': 'application/json' },
       body: JSON.stringify({ code: 'SUCCESS', message: '成功' })
     }
     ```
   - 失败时返回：
     ```js
     return {
       statusCode: 200, // 微信要求即使失败也返回 200，避免重复通知
       headers: { 'content-type': 'application/json' },
       body: JSON.stringify({ code: 'FAIL', message: '错误说明' })
     }
     ```

### 五、前端：统一支付入口设计

1. **新增统一支付方法 `payForOrder`**
   - 位置建议：新建 `store/pay.js` 或在 `store/wallet.js` 中扩展。
   - 方法签名：`payForOrder({ method, orderId, amount })`
   - 逻辑：
     - `method === 'balance'`：
       - 调用 `walletStore.pay(amount, orderId)`，成功后更新订单支付状态。
     - `method === 'wechat'`：
       1. 确保已通过 `order-service` 创建订单，状态为 `pending_pay`。
       2. 调用云对象 `paymentService.createJsapiOrder({ orderId })` 获取支付参数。
       3. 使用 `wx.requestPayment` 发起支付：
          - 传入 `timeStamp/nonceStr/package/signType/paySign`。
       4. 在 `success` 回调中提示“支付处理中”，并可轮询 `paymentService.queryOrder` 直到状态为成功或失败。
       5. 返回 `{ success, reason }` 给调用页面。
2. **改造下单页面**
   - `pages/client/forms/pickup.vue` / `errand.vue` 等：
     - 继续使用 `order-service` 创建订单（初始 `pay_status = 'unpaid'` / `status = 'pending_pay'`）。
     - 余额支付时调用 `payForOrder({ method: 'balance', orderId, amount })`。
     - 微信支付时调用 `payForOrder({ method: 'wechat', orderId, amount })`，不再使用现在的“模拟微信支付 toast”。

### 六、测试与上线 Checklist

1. **联调测试**
   - 使用微信开发者工具 + 测试商户号，发起小额支付（如 0.01 元）。
   - 验证：
     - `createJsapiOrder` 返回 `prepay_id` 与支付参数。
     - 小程序能成功拉起微信收银台。
     - 支付成功后，HTTP 回调被触发，本地订单与交易流水状态更新正确。
2. **异常场景测试**
   - 用户主动取消支付。
   - 回调重复通知（幂等逻辑有效）。
   - 签名错误、金额不一致等异常能被拒绝并记录日志。
3. **上线前核查**
   - 商户平台回调 URL 已指向线上的 HTTP 云函数地址。
   - 生产环境的 `mchid/appid/APIv3 Key/serial_no/私钥` 全部为正式配置，并妥善保管。
