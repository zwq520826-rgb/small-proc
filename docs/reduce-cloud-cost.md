# 阿里云计费下的云函数/云对象减费方案（结合当前项目）

## 快速减费清单（官方建议对应）
- **删除未使用的定时任务**：控制台检查并移除 `uni-stat-cron`、`uni-analyse-searchhot` 等默认定时器，避免每天的保底费用。
- **合并函数/云对象，减少实例数**：将多个业务函数合并为单个云对象或通过路由集中处理，实例数越少，保底费用越低。
- **精简第三方/辅助云函数**：移除不用的示例/模板函数，减少部署数量。

## 针对本项目的具体建议
1) **定时任务清理**
   - 打开 uniCloud 控制台（阿里云空间）> 云函数与云对象 > 定时任务，确认是否存在默认的 `uni-stat-cron`、`uni-analyse-searchhot` 或其他未使用的 cron，直接删除。
   - 若后续需要统计/热搜，再按需开启，默认保持关闭。

2) **合并业务云对象，减少保底费用**
   - 当前阿里云目录下有多个云对象：`order-service`、`rider-service`、`wallet-service`、`payment-service`、`admin-service`、`address-service`、`config-service`，每个都会产生单独的保底计费。
   - 建议将低频/小功能合并：
     - 将 `address-service`、`config-service` 入口合并进 `order-service`（新增 method），避免单独实例。
     - 评估将 `wallet-service` 的查询/充值/提现与骑手结算相关接口并入 `rider-service`，保持一个“骑手域”云对象。
     - 如后台管理接口量不大，可将 `admin-service` 合入 `order-service` 或新增单一 `admin` 云对象集中管理。
   - 合并方式：保留一个云对象的 `index.obj.js`，在 `module/` 下增加对应方法并调整前端调用路径；删除被合并的云对象目录，重新部署。

3) **避免多余触发与频繁调用**
   - 前端调用缓存：
     - 登录态/配置等结果写入本地缓存（本项目已在多处使用 `loadFromStorage`，保持并避免重复 onShow 拉取）。
     - 列表/统计接口增加最小请求间隔（如 5~10 秒）或分页增量拉取，减少高频刷新。
   - 轮询改推送/订阅：若有前端轮询订单状态，优先使用数据库触发器或手动下发更新，减少空调用。

4) **清理未用的页面或功能调用**
   - 已移除的用户钱包前端页面与 store 不再调用 `wallet-service`，后续可按合并策略取消独立部署，避免保底费用。
   - 检查其他“待实现/示例”入口，避免误触发云端调用。

5) **部署与监控**
   - 部署前检查 `uniCloud-aliyun/cloudfunctions` 目录，确保仅保留需要的云对象；合并后删除冗余目录再上传。
   - 使用 uniCloud 控制台的调用次数/费用报表，定位高频函数，按上面策略合并或限频。

## 执行顺序建议
1. 控制台删除未用定时任务。
2. 选定合并策略（先合并 `address-service`、`config-service` 到 `order-service`）。
3. 更新前端/后端调用路径，删除被合并函数目录后重新部署。
4. 观察 1~2 天费用报表，继续合并低频函数或优化调用频率。
