## 骑手端实名认证（手机号实名核验）接入计划

> 目标：在现有的 **`rider-service` 云对象**中集成 uniCloud「手机号实名核验」能力，使用**三要素（详版）**对骑手进行实名认证（姓名 + 身份证号 + 手机号）。在骑手提交认证申请时，自动进行实名核验，只有通过核验才能完成认证。

---

### 一、你需要先完成的准备工作

1. **开通 uniCloud 手机号实名核验服务**
   - 登录 [uniCloud 控制台](https://unicloud.dcloud.net.cn/)
   - 进入【扩展能力】→ 找到「uni手机号实名核验」
   - 确认已开通该服务（可能需要付费，具体费用请查看控制台）
   - 确认服务空间已绑定并激活

2. **确认本项目使用的认证类型**
   - 本项目仅使用：**三要素认证（详版）**（姓名 + 身份证号 + 手机号）
   - 在骑手提交认证申请时自动调用

---

### 二、在现有 `rider-service` 中添加扩展库依赖

### 三、修改 `rider-service` 云对象：集成实名认证

> 文件：`uniCloud-aliyun/cloudfunctions/rider-service/index.obj.js`

#### 3.1 在文件顶部引入扩展库

在文件开头（`const uniID = require('uni-id-common')` 之后）添加：

```javascript
const verify = uniCloud.importObject('uni-cloud-verify')
```

#### 3.2 修改 `submitApplication` 方法

在 `submitApplication` 方法中，**在保存数据到数据库之前**，先调用实名认证接口进行验证。

**修改位置**：在验证短信验证码之后、保存数据之前，添加实名认证逻辑：

```javascript
async submitApplication(data) {
  const auth = checkAuth(this)
  if (auth.code) return auth
  const uid = auth.uid

  const {
    name,
    student_no,
    college_class,
    id_card,
    mobile,
    smsCode
  } = data || {}

  if (!name || !student_no || !college_class || !id_card || !mobile || !smsCode) {
    return {
      code: 'INVALID_PARAM',
      message: '请完整填写认证信息并输入验证码'
    }
  }

  // 校验短信验证码
  const now = Date.now()
  const verifyCollection = db.collection('opendb-verify-codes')
  const dbCmd = db.command
  
  const codeRecord = await verifyCollection.where({
    mobile: mobile.trim(),
    scene: 'bind-mobile-by-sms',
    code: smsCode,
    state: 0, // 未使用
    expired_date: dbCmd.gt(now) // 未过期
  }).limit(1).get()
  
  if (codeRecord.data.length === 0) {
    return {
      code: 'INVALID_VERIFY_CODE',
      message: '验证码错误或已过期'
    }
  }
  
  // 标记验证码为已使用
  await verifyCollection.doc(codeRecord.data[0]._id).update({
    state: 1
  })

  // ========== 新增：实名认证核验 ==========
  try {
    // 获取认证管理对象
    const verifyManager = verify.getVerifyManager()
    
    // 调用三要素（详版）认证接口
    const verifyResult = await verifyManager.verifyThreeFactorDetail({
      name: name.trim(),
      idCard: id_card.trim(),
      mobile: mobile.trim()
    })

    // 检查认证结果
    if (!verifyResult || !verifyResult.verifyResult) {
      return {
        code: 'VERIFY_FAILED',
        message: verifyResult?.message || '实名认证失败，请检查姓名、身份证号和手机号是否匹配'
      }
    }
  } catch (error) {
    console.error('实名认证接口调用失败:', error)
    return {
      code: 'VERIFY_ERROR',
      message: '实名认证服务异常，请稍后重试'
    }
  }
  // ========== 实名认证核验结束 ==========

  // 继续原有的保存逻辑
  const col = db.collection('rider_profiles')
  const existed = await col.where({ user_id: uid }).limit(1).get()
  
  if (existed.data.length) {
    const id = existed.data[0]._id
    await col.doc(id).update({
      name,
      student_no,
      college_class,
      id_card,
      mobile,
      status: 'approved',
      real_name_verified: true,        // 新增：标记已通过实名认证
      real_name_verify_time: now,      // 新增：实名认证时间
      update_time: now
    })
    return {
      code: 0,
      message: '已更新认证资料，已通过认证',
      data: { _id: id }
    }
  } else {
    const addRes = await col.add({
      user_id: uid,
      name,
      student_no,
      college_class,
      id_card,
      mobile,
      status: 'approved',
      real_name_verified: true,        // 新增：标记已通过实名认证
      real_name_verify_time: now,      // 新增：实名认证时间
      create_time: now,
      update_time: now
    })
    return {
      code: 0,
      message: '认证已通过',
      data: { _id: addRes.id }
    }
  }
}
```

### 四、扩展数据库 Schema：添加实名认证字段

> 文件：`uniCloud-aliyun/database/rider_profiles.schema.json`

在 `properties` 中添加以下字段（在 `update_time` 之后）：

```json
{
  "real_name_verified": {
    "bsonType": "bool",
    "description": "是否已通过实名认证",
    "default": false
  },
  "real_name_verify_time": {
    "bsonType": "timestamp",
    "description": "实名认证通过时间"
  },
  "real_name_verify_message": {
    "bsonType": "string",
    "description": "实名认证结果说明（可选，用于记录详细信息）"
  }
}
```

**注意**：这些字段不是必填项（不在 `required` 数组中），因为可能存在历史数据。

### 五、前端页面：无需修改（已满足需求）

现有的 `pages/rider/verify.vue` 页面已经收集了所需信息（姓名、身份证号、手机号），提交时会自动调用 `riderService.submitApplication`，现在会自动进行实名认证。

**可选优化**：可以在提交时显示"正在验证身份信息..."的提示：

```javascript
const submit = async () => {
  uni.showLoading({ title: '正在验证身份信息...', mask: true })
  try {
    const res = await riderService.submitApplication(form.value)
    uni.hideLoading()
    if (res.code === 0) {
      uni.showToast({ title: res.message || '认证成功', icon: 'success' })
      await loadProfile()
    } else {
      // 实名认证失败的特殊提示
      if (res.code === 'VERIFY_FAILED') {
        uni.showToast({ 
          title: res.message || '实名认证失败，请检查信息是否正确', 
          icon: 'none',
          duration: 3000
        })
      } else {
        uni.showToast({ title: res.message || '提交失败', icon: 'none' })
      }
    }
  } catch (e) {
    uni.hideLoading()
    uni.showToast({ title: '提交失败', icon: 'none' })
  }
}
```

### 六、业务流程说明

1. **骑手填写认证信息**
   - 在 `pages/rider/verify.vue` 页面填写：姓名、学号、学院班级、身份证号、手机号
   - 获取短信验证码并填写

2. **提交认证申请**
   - 前端调用 `riderService.submitApplication(formData)`
   - 云对象先校验短信验证码（现有逻辑）

3. **自动进行实名认证**（新增）
   - 调用 `uni-cloud-verify` 的三要素（详版）接口
   - 验证姓名、身份证号、手机号是否匹配
   - 如果验证失败，直接返回错误，不保存数据

4. **保存认证资料**
   - 只有实名认证通过后，才保存到 `rider_profiles` 集合
   - 同时标记 `real_name_verified = true` 和记录认证时间

5. **结果反馈**
   - 认证成功：返回成功消息，更新页面状态
   - 认证失败：返回具体错误信息，提示用户检查信息
