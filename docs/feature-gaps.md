# 功能缺口建议（small-proc 与 small-admin）

## small-admin（运营后台）
- **骑手流水明细**：已新增按用户查询流水，建议补充导出 CSV/筛选时间范围，并在提现列表中可点击查看关联流水。
- **订单/流水对账**：提供按日聚合的“骑手收入 vs 流水 vs 提现”报表，便于审计差异。
- **异常监控面板**：展示云函数错误次数、余额负数/退款异常订单提示，减少人工排查。
- **权限与操作日志**：记录提现审核、订单修改等敏感操作，后续出现纠纷可追溯。
- **内容配置管理**：首页公告/广告位已有，补充“客服电话/运营联系方式”配置，避免写死。

## small-proc（用户/骑手端）
- **用户端钱包入口控制**：已做“仅骑手可用”提示，可考虑在用户端隐藏入口并在订单支付界面提示余额不可用，避免误点。
- **缓存与降频**：对首页内容、价格配置等接口增加本地缓存 TTL，减少频繁云调用（已部分实现，可统一封装）。
- **订单状态推送优化**：如有轮询，建议改用数据库触发/订阅或限频策略，降低调用量。a d mi
- **异常兜底**：在取单/送达等关键操作失败时，统一错误码与提示，便于定位云端问题。
- **运行时拨打/复制分流**：目前客户电话改为拨打，可在非微信端降级为复制，避免失败体验。

## 云端与费用优化
- **云对象合并**：address/config 已合并进 order-service；可继续评估将 wallet 与骑手结算接口合并，减少实例数。
- **定时任务清理**：控制台删除 `uni-stat-cron`、`uni-analyse-searchhot` 等未用 cron。
- **读写限频**：为列表/统计接口增加最小间隔或“加载更多”模式，避免高频重复调用。

## 后续落地建议
1) 部署最新 order-service（合并 address/config）并删除旧实例。
2) 在后台页面补充流水导出/时间筛选；前端按需隐藏用户钱包入口。
3) 清理云端未用定时任务，观察 1~2 天费用报表后再决定进一步合并。
