# 数据库集合说明文档

> 本文档说明校园跑腿小程序项目中所有数据库集合的作用和字段含义。

---

## 一、核心业务集合

### 1. `orders` - 订单表

**作用**：存储所有订单/任务数据，用户端和骑手端共用。

**权限说明**：
- 读取：订单的下单用户或接单骑手可以查看
- 创建：已登录用户可以创建
- 更新：订单的下单用户或接单骑手可以更新
- 删除：只有下单用户可以删除

**字段说明**：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `_id` | string | 是 | 订单ID，系统自动生成 |
| `user_id` | string | 是 | 下单用户ID，关联 `uni-id-users._id` |
| `rider_id` | string | 否 | 接单骑手ID（可选），关联 `uni-id-users._id` |
| `type` | string | 是 | 订单类型，枚举值：<br>- `pickup`：快递代取<br>- `errand`：跑腿服务<br>- `post`：帮我寄件<br>- `print`：打印服务 |
| `type_label` | string | 否 | 订单类型标签（显示名称） |
| `status` | string | 是 | 订单状态，枚举值：<br>- `pending_accept`：待接单<br>- `pending_pickup`：待取货<br>- `delivering`：配送中<br>- `completed`：已完成<br>- `cancelled`：已取消 |
| `price` | double | 是 | 订单金额（≥0） |
| `pickup_location` | string | 否 | 取件地址 |
| `delivery_location` | string | 否 | 送达地址 |
| `address` | string | 否 | 收货地址（兼容字段） |
| `content` | object | 否 | 订单内容详情，包含：<br>- `description`：订单描述<br>- `images`：图片URL数组<br>- `phone`：联系电话<br>- `pickup_images`：取件凭证图片数组<br>- `delivery_images`：送达凭证图片数组<br>- `runner_fee`：跑腿费<br>- `is_urgent`：是否加急（bool）<br>- `is_delivery`：是否送货上门（bool）<br>- `dorm_number`：寝室号 |
| `tags` | array | 否 | 订单标签数组 |
| `pickup_distance` | double | 否 | 取件距离（公里） |
| `create_time` | timestamp | 否 | 创建时间（自动生成） |
| `accept_time` | timestamp | 否 | 接单时间 |
| `pickup_time` | timestamp | 否 | 取货时间 |
| `complete_time` | timestamp | 否 | 完成时间 |

**业务说明**：
- 订单创建时，`status` 为 `pending_accept`，`rider_id` 为空
- 骑手接单后，`status` 更新为 `pending_pickup`，`rider_id` 填入接单骑手ID
- 骑手确认取货后，`status` 更新为 `delivering`
- 骑手确认送达后，`status` 更新为 `completed`

### 2. `rider_profiles` - 骑手认证表

**作用**：存储骑手的认证信息，用于判断用户是否是骑手。

**权限说明**：
- 读取：用户只能查看自己的认证信息
- 创建：已登录用户可以创建
- 更新：用户只能更新自己的认证信息
- 删除：不允许删除

**字段说明**：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `_id` | string | 是 | 认证记录ID，系统自动生成 |
| `user_id` | string | 是 | 用户ID，关联 `uni-id-users._id` |
| `name` | string | 是 | 姓名 |
| `student_no` | string | 是 | 学号 |
| `college_class` | string | 是 | 学院+班级，如：计算机学院 21级计科1班 |
| `id_card` | string | 是 | 身份证号 |
| `mobile` | string | 是 | 手机号（已通过短信验证） |
| `status` | string | 是 | 审核状态，枚举值：<br>- `pending`：待审核<br>- `approved`：已通过（表示是有效骑手）<br>- `rejected`：已拒绝 |
| `remark` | string | 否 | 审核备注（管理员填写） |
| `create_time` | timestamp | 否 | 创建时间（自动生成） |
| `update_time` | timestamp | 否 | 最近更新时间 |
| `real_name_verified` | bool | 否 | 是否已通过实名认证（默认 false） |
| `real_name_verify_time` | timestamp | 否 | 实名认证通过时间 |
| `real_name_verify_message` | string | 否 | 实名认证结果说明（可选） |

**业务说明**：
- 判断用户是否是骑手：查询 `rider_profiles` 集合，`user_id` 匹配且 `status === 'approved'`
- 当前实现中，提交认证后直接设置为 `approved`（已取消审核流程）
- 提交认证时会自动进行实名认证（三要素：姓名+身份证号+手机号）

### 3. `wallets` - 钱包表

**作用**：存储用户钱包余额和统计信息。

**权限说明**：
- 读取：用户只能查看自己的钱包信息
- 创建/更新/删除：不允许直接操作（由云函数管理）

**字段说明**：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `_id` | string | 是 | ID，系统自动生成 |
| `user_id` | string | 是 | 用户ID，关联 `uni-id-users._id` |
| `balance` | double | 是 | 账户余额（≥0，默认 0） |
| `frozen_balance` | double | 否 | 冻结金额（用于提现中的金额，≥0，默认 0） |
| `total_income` | double | 否 | 累计收入（骑手收入，≥0，默认 0） |
| `total_expense` | double | 否 | 累计支出（≥0，默认 0） |
| `create_time` | timestamp | 否 | 创建时间 |
| `update_time` | timestamp | 否 | 更新时间 |

**业务说明**：
- 钱包记录由云函数自动创建和管理
- `balance` 为可用余额，`frozen_balance` 为冻结金额（如提现中）
- `total_income` 主要用于统计骑手收入
- 所有金额变动都会记录到 `transactions` 集合

---

### 4. `transactions` - 交易记录表

**作用**：记录所有资金变动流水，用于对账和审计。

**权限说明**：
- 读取：用户只能查看自己的交易记录
- 创建/更新/删除：不允许直接操作（由云函数管理）

**字段说明**：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `_id` | string | 是 | ID，系统自动生成 |
| `user_id` | string | 是 | 用户ID，关联 `uni-id-users._id` |
| `type` | string | 是 | 交易类型，枚举值：<br>- `recharge`：充值<br>- `withdraw`：提现<br>- `pay`：支付<br>- `income`：收入（骑手接单收入）<br>- `refund`：退款 |
| `amount` | double | 是 | 交易金额 |
| `balance_before` | double | 否 | 交易前余额 |
| `balance_after` | double | 否 | 交易后余额 |
| `order_id` | string | 否 | 关联订单ID（可选），关联 `orders._id` |
| `status` | string | 是 | 交易状态，枚举值：<br>- `pending`：处理中<br>- `success`：成功<br>- `failed`：失败 |
| `remark` | string | 否 | 备注 |
| `create_time` | timestamp | 否 | 创建时间 |

**业务说明**：
- 所有资金变动都会创建交易记录
- `type` 为 `income` 时，通常关联 `order_id`（骑手完成订单后的收入）
- `type` 为 `pay` 时，通常关联 `order_id`（用户支付订单）
- `balance_before` 和 `balance_after` 用于记录余额变化，便于对账

---

### 5. `addresses` - 地址表

**作用**：存储用户的收货地址信息。

**权限说明**：
- 读取/创建/更新/删除：当前 schema 中权限设置为 false（实际由云函数控制）

**字段说明**（根据代码实现推断）：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `_id` | string | 是 | ID，系统自动生成 |
| `user_id` | string | 是 | 用户ID，关联 `uni-id-users._id` |
| `name` | string | 是 | 收件人姓名 |
| `phone` | string | 是 | 手机号 |
| `detail` | string | 是 | 详细地址 |
| `school_area` | string | 否 | 校区 |
| `is_default` | bool | 否 | 是否默认地址 |
| `create_time` | timestamp | 否 | 创建时间 |

**业务说明**：
- 每个用户可以添加多个地址
- 只能有一个默认地址（`is_default = true`）
- 设置新地址为默认时，会自动取消其他默认地址

## 二、系统集合（uni-id 相关）

### 6. `uni-id-users` - 用户表

**作用**：存储用户基本信息（由 uni-id 模块管理）。

**主要字段**：
- `_id`：用户ID（主键）
- `username`：用户名
- `nickname`：昵称
- `mobile`：手机号
- `email`：邮箱
- `avatar`：头像
- `wx_openid`：微信 openid（小程序登录用）
- `role`：角色数组
- `register_date`：注册时间
- 等...

**说明**：此集合由 uni-id 模块管理，不建议直接操作。

---

### 7. `opendb-verify-codes` - 验证码表

**作用**：存储短信验证码（由 uni-captcha 模块管理）。

**主要字段**：
- `mobile`：手机号
- `code`：验证码
- `scene`：场景（如：`bind-mobile-by-sms`）
- `state`：状态（0：未使用，1：已使用）
- `expired_date`：过期时间

**说明**：此集合由 uni-captcha 模块管理，用于短信验证码校验。

## 三、集合关系图

```
uni-id-users (用户表)
    ├── orders (订单表) - user_id, rider_id
    ├── rider_profiles (骑手认证表) - user_id
    ├── wallets (钱包表) - user_id
    ├── transactions (交易记录表) - user_id
    └── addresses (地址表) - user_id

orders (订单表)
    └── transactions (交易记录表) - order_id
```

## 四、常用查询示例

### 判断用户是否是骑手

```javascript
const riderProfile = await db.collection('rider_profiles')
  .where({
    user_id: uid,
    status: 'approved'
  })
  .limit(1)
  .get()

const isRider = riderProfile.data.length > 0
```

### 查询用户的订单列表

```javascript
const orders = await db.collection('orders')
  .where({
    user_id: uid
  })
  .orderBy('create_time', 'desc')
  .get()
```

### 查询骑手接单的任务列表

```javascript
const tasks = await db.collection('orders')
  .where({
    rider_id: uid,
    status: db.command.in(['pending_pickup', 'delivering'])
  })
  .orderBy('accept_time', 'desc')
  .get()
```

### 查询待接单的订单（大厅任务）

```javascript
const hallTasks = await db.collection('orders')
  .where({
    status: 'pending_accept',
    rider_id: db.command.exists(false) // 或 rider_id: null
  })
  .orderBy('create_time', 'desc')
  .get()
```

## 五、注意事项

1. **权限控制**：所有集合都配置了权限规则，确保用户只能操作自己的数据
2. **外键关联**：`user_id` 和 `rider_id` 都关联 `uni-id-users._id`
3. **时间字段**：大部分集合都有 `create_time` 和 `update_time` 字段
4. **状态枚举**：订单状态、交易状态、审核状态等都使用枚举值，确保数据一致性
5. **金额字段**：所有金额字段类型为 `double`，且设置了最小值限制（≥0）
