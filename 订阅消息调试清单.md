# 订阅消息调试检查清单

## 如果测试后还是收不到消息，请检查：

### 1. 检查订阅消息模板ID是否正确

在微信公众平台：
- 功能 → 订阅消息 → 我的模板
- 确认模板ID与数据库中的一致

当前配置的模板ID：

- 送达通知：d-epVcwVVoWk9ZuzGPUVUeE4Uy3DgixAEYetadc-2U8

### 2. 检查数据库配置

在 uniCloud 控制台查询 maintenance_settings 表：
```javascript
db.collection('maintenance_settings')
  .where({ _id: 'order_subscribe_notify' })
  .get()
```

确认：
- enable: true
- template_accept_id 和 template_deliver_id 有值

### 3. 查看云函数日志

在 uniCloud 控制台：
- 云函数 → 日志
- 筛选 order-service
- 查找 "sendSubscribeMessage" 相关日志
- 查看是否有错误信息

### 4. 检查用户 openid

查询用户表：
```javascript
db.collection('uni-id-users')
  .where({ _id: '你的用户ID' })
  .field({ wx_openid: true })
  .get()
```

确认 wx_openid.mp 字段有值

### 5. 检查订单的订阅授权记录

查询订单：
```javascript
db.collection('orders')
  .orderBy('create_time', 'desc')
  .limit(1)
  .get()
```

检查 content.subscribe_notify 字段：
- tpl_accept_accept: 应该为 true
- tpl_deliver_accept: 应该为 true
- last_error: 如果有值，说明发送失败

### 6. 常见错误码对照

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| 40001 | access_token 无效 | AppSecret 配置错误，重新检查 |
| 40003 | openid 无效 | 用户未正确登录 |
| 43101 | 用户拒绝授权 | 重新下单并授权 |
| 47003 | 模板参数错误 | 检查模板ID和参数格式 |
| 43104 | 订阅关系失效 | 用户需要重新授权 |

## 快速验证 AppSecret 是否生效

可以在云函数中添加测试代码：

```javascript
// 在 order-service 中添加测试方法
async testWechatConfig() {
  const createConfig = require('uni-config-center')
  const center = createConfig({ pluginId: 'uni-id' })
  const cfg = center.config() || {}
  const mpCfg = cfg['mp-weixin'] || {}
  const oauth = mpCfg.oauth || {}
  const wx = oauth.weixin || {}
  
  console.log('AppID:', wx.appid)
  console.log('AppSecret 是否配置:', !!wx.appsecret)
  console.log('AppSecret 长度:', wx.appsecret ? wx.appsecret.length : 0)
  
  return {
    code: 0,
    data: {
      hasAppid: !!wx.appid,
      hasAppsecret: !!wx.appsecret,
      appsecretLength: wx.appsecret ? wx.appsecret.length : 0
    }
  }
}
```

然后在前端调用测试。
