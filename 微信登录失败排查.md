# 微信登录失败排查步骤

## 当前情况
- AppSecret 已更新到配置文件
- 云函数已上传
- 但登录仍然失败："获取第三方账号失败"

## 立即尝试的解决方案

### 方案1：在 uniCloud Web 控制台检查配置

1. 打开浏览器，访问 [uniCloud 控制台](https://unicloud.dcloud.net.cn/)
2. 选择你的服务空间
3. 左侧菜单：**云函数** → 找到 `uni-id-co`
4. 点击 **云函数详情** → **配置**
5. 查看是否有环境变量或配置项
6. 点击 **重启** 云函数（让新配置生效）

### 方案2：查看云函数日志

1. 在 uniCloud 控制台
2. 左侧菜单：**云函数** → **日志**
3. 筛选 `uni-id-co` 云函数
4. 查看最近的错误日志
5. 找到具体的错误信息（可能是 40001、40003 等错误码）

### 方案3：强制重新上传（推荐）

在 HBuilderX 中：

1. **删除云端的 uni-id-co**
   - 右键点击 `uni_modules/uni-id-pages/uniCloud/cloudfunctions/uni-id-co`
   - 选择 **删除云端云函数**

2. **重新上传**
   - 右键点击 `uni-id-co`
   - 选择 **上传部署**
   - 等待上传完成

3. **清除小程序缓存**
   - 微信开发者工具：工具 → 清除缓存 → 清除全部缓存
   - 重新编译运行

### 方案4：检查微信开放平台配置

可能不是 AppSecret 的问题，而是其他配置：

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 开发 → 开发管理 → 开发设置
3. 检查 **服务器域名** 是否配置了 uniCloud 的域名：
   - request合法域名：需要添加你的 uniCloud 服务空间域名
   - 例如：`https://xxx.bspapp.com`

4. 检查 **业务域名** 是否配置

### 方案5：临时测试 - 使用手机号登录

如果微信登录一直有问题，可以先让用户使用其他登录方式：

1. 在 `uni_modules/uni-id-pages/config.js` 中
2. 修改 `loginTypes` 数组，添加其他登录方式：
   ```javascript
   loginTypes: [
     'weixin',
     'smsCode'  // 添加短信验证码登录
   ]
   ```

## 最可能的原因

根据经验，"获取第三方账号失败"通常是：

1. **AppSecret 配置未生效** - 云函数缓存了旧配置
2. **微信接口调用失败** - access_token 获取失败
3. **用户 openid 获取失败** - 微信授权流程问题
