# 校园跑腿小程序 - 数据库与云函数开发计划

## 📋 需要创建的数据库表

### 1. 核心业务表

#### `orders` - 订单表（用户端和骑手端共用）

**用途**：存储所有订单/任务数据

**关键字段**：
- `user_id` - 下单用户ID
- `rider_id` - 接单骑手ID（可选）
- `type` - 订单类型：`pickup`（快递代取）、`errand`（跑腿服务）、`post`（帮我寄件）、`print`（打印服务）
- `status` - 订单状态：`pending_accept`（待接单）、`delivering`（配送中）、`completed`（已完成）、`cancelled`（已取消）
- `price` - 订单金额
- `pickup_location` - 取件地址
- `delivery_location` - 送达地址
- `content` - 订单详情（描述、图片、电话等）

#### `addresses` - 地址表

**用途**：存储用户收货地址

**关键字段**：
- `user_id` - 用户ID
- `name` - 收货人姓名
- `phone` - 联系电话
- `school_area` - 校区
- `detail` - 详细地址
- `is_default` - 是否默认地址

#### `wallets` - 钱包表

**用途**：存储用户钱包余额

**关键字段**：
- `user_id` - 用户ID
- `balance` - 余额
- `frozen_balance` - 冻结金额（可选）

#### `transactions` - 交易记录表

**用途**：记录所有资金变动

**关键字段**：
- `user_id` - 用户ID
- `type` - 交易类型：`recharge`（充值）、`pay`（支付）、`income`（收入）、`refund`（退款）
- `amount` - 金额
- `order_id` - 关联订单ID（可选）
- `status` - 交易状态

### 2. 扩展功能表（可选）

#### `coupons` - 优惠券表（如果实现优惠券功能）

#### `messages` - 消息通知表（如果实现消息功能）

---

## 🔧 需要创建的云函数（云对象）

### 1. `order-service` - 订单服务（核心）

包含用户端和骑手端的所有订单相关功能：

**用户端功能**：
- `createOrder` - 创建订单
- `getOrderList` - 获取我的订单列表
- `getOrderDetail` - 获取订单详情
- `cancelOrder` - 取消订单
- `deleteOrder` - 删除订单

**骑手端功能**：
- `getHallTasks` - 获取大厅任务列表（待接单的订单）
- `grabTask` - 抢单（接单）
- `getMyTasks` - 获取我的任务列表
- `confirmPickup` - 确认取货
- `confirmDelivery` - 确认送达

### 2. `wallet-service` - 钱包服务

- `getBalance` - 获取余额
- `recharge` - 充值
- `pay` - 支付订单
- `getTransactionList` - 获取交易记录
- `refund` - 退款（订单取消时）

### 3. `address-service` - 地址服务

- `getAddressList` - 获取地址列表
- `addAddress` - 添加地址
- `updateAddress` - 更新地址
- `deleteAddress` - 删除地址
- `setDefaultAddress` - 设置默认地址

## ⚠️ 注意事项

### 1. 订单和任务共用 `orders` 表

- **用户端**：查询 `user_id == 当前用户` 的订单
- **骑手端大厅**：查询 `status == 'pending_accept'` 且 `rider_id == null` 的订单
- **骑手端我的任务**：查询 `rider_id == 当前用户` 的订单

### 2. 权限控制

- 每个 Schema 都要配置 `permission` 规则
- 云对象中也要做权限校验
- 使用 `this.getUniIdToken()` + `uni-id-common.checkToken()` 获取用户身份

### 3. 数据关联

- `orders.user_id` 关联 `uni-id-users._id`
- `orders.rider_id` 关联 `uni-id-users._id`
- `transactions.order_id` 关联 `orders._id`

### 4. 云对象开发要点

- 必须引入 `uni-id-common` 来验证 token
- 使用 `this.getUniIdToken()` 获取客户端传来的 token
- 通过 `uniIdCommon.checkToken(token)` 验证并获取 `uid`
- 所有数据操作都要绑定 `user_id: uid`
- 查询时自动过滤，只返回当前用户的数据

## 📝 开发流程

```
创建数据库 Schema → 上传 Schema → 创建云对象 → 上传云对象 → 前端调用
```

1. **创建数据库 Schema**：在 `uniCloud-aliyun/database/` 创建 `.schema.json` 文件
2. **上传 Schema**：在 HBuilderX 中右键 schema 文件，选择"上传 DB Schema"
3. **创建云对象**：在 `uniCloud-aliyun/cloudfunctions/` 创建云对象目录
4. **上传云对象**：右键云函数目录，选择"上传部署"
5. **前端调用**：使用 `uniCloud.importObject('order-service')` 调用
